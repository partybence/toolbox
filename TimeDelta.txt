Stores and displays a difference in time

Instantiating:
	> new TimeDelta(millis:Number)
	> new TimeDelta(timedelta:String)
		String timedelta can be provided in the following format: ([sign]<number>[unit])+
			where units can be (d)ays, (h)ours, (m)inutes, (s)econds or nothing for millis.
			Optional space separators between number and unit, and space or comma separators between groups are allowed.
			When using negative signs, one sign affects one group only (next group will default in adding to the current delta).
			
	Examples:
		> new TimeDelta(3600000) // 1 hour
		> new TimeDelta("1d 1h, -20m,30s") // 1 daz, 40 minutes (1 hour - 20 minutes) and 30 seconds
		
Functions:
	.Days() .Hours() .Minutes() .Seconds() .Millis()
		These functions return remaining values that can't be further devided into a larger unit, rounded down.
		Example:
			> (new TimeDelta("1d 2h 30m")).Hours()
			< 2
	
	.TotalDays() .TotalHours() .TotalMinutes() .TotalSeconds() .TotalMillis()
		These functions return exactly how many of a unit are in the TimeDelta.
		Example:
			> (new TimeDelta("1d 2h 30m")).TotalHours()
			< 26.5
	
	.Format(format)
		This function can be used to easily stringify the delta in a desired format.
			The format can use the following sequences:
				%% for literal "%"
				%d %h %m %s %. for accessing .Days() .Hours() .Minutes() .Seconds() and .Millis() in order
				%D %H %M %S %: for accessing .TotalDays() .TotalHours() .TotalMinutes() .TotalSeconds() and .TotalMillis() in order
				and the letters preceded by 0 for rounding down and padding with 0 to a length of 3 for millis and a length of 2 for any other.
		Examples:
			> (new TimeDelta("1d 2h 30m")).Format("%d day(s), %0h:%0m")
			< "1 day(s), 02:30"
			
			> (new TimeDelta("1d 2h 9m")).Format("%0H hour(s) %m minute(s)")
			< "26 hour(s) 9 minute(s)"
